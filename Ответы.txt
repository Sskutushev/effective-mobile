2. Расскажите, что такое Promise в JavaScript, как он работает, какие
основные состояния у него бывают? Каким еще способомможно
обработать асинхронный код? Какую роль в асинхронности играет
событийный цикл (Event Loop)?

Промисы это объекты(обещания) асинхронного результата. 
Если он уже "решился", его нельзя отменить или передумать, состояние фиксируется намертво. 
Три состояния: Pending (ждем), Fulfilled (успех) и Rejected (ошибка). 
Обработка идет через .then, .catch и .finally, но по факту сейчас все юзают async/await, это просто сахар над промисами, 
чтобы код был плоским и чистым. Цикл событий это механизм который разруливает выполнение кода и позволяет JS не блокировать поток на тяжелых задачах. 
Его главная работа постоянно чекать стек вызовов (Call Stack). Когда стек пустой, он закидывает туда задачи из очередей. У промисов приоритет выше. 
Пока очередь микрозадач не пуста, цикл событий не перейдет к обычным задачам


3. Что такое ООП, и какие его ключевые принципы вы могли бы
описать? При объяснении приводите примеры. Максимально
подробно расскажите о том, как ООП реализовано в JavaScript.

ООП это подход в программировании где представляем все в виде объектов которые взаимодействуют между собой, 
помогает собирать код как конструктор из логических кирпичей. 
Основные принципы: инкапсуляция (прячем логику внутри, например баланс кошелька меняем только через метод), 
наследование (создаем новый класс на базе старого), полиморфизм (один метод работает по разному для разных типов) 
и абстракция (скрываем детали реализации оставляя только важное).
В JS ООП реализовано специфично через прототипное наследование
Когда ищем свойство у объекта JS сначала смотрит у него самого, потом лезем в его прототип и так далее по цепочке. 
Синтаксис class это просто сахар над прототипами, под капотом всё равно остаются функции-конструкторы и объекты. 
Классы делают код удобнее и читабельнее для тех кто пришел из других языков.


4. Опишите максимально детально, как браузер обрабатывает URL,
начиная с момента его ввода в адресную строку и заканчивая
отображением страницы. Какие технологии позволяют ускорить
процесс? Какие проблемы (безопастность, междоменное
взаимодействие) могут возникать в процессе?

Когда ты вводишь URL, браузер должен превратить текст в пиксели. Вот как это происходит:
1. навигация и DNS
2. запрос и ответ (HTTP)
3. Критический путь рендеринга

Все начинается с DNS lookup где браузер ищет IP адрес домена потом ставится TCP соединение и TLS рукопожатие для HTTPS. 
Дальше сервер отдает HTML и браузер запускает Critical Rendering Path строит DOM и CSSOM деревья объединяет их в Render Tree считает геометрию в Layout и рисует пиксели в Paint. 
Чтобы ускорить процесс используют протоколы HTTP/2 или HTTP/3 QUIC для мультиплексирования и CDN чтобы забирать статику с ближайшего к юзеру сервера. 
Также оптимизируют рендеринг через минификацию JS CSS сжатие картинок WebP и использование Vite для раздачи модулей на лету и кэширования зависимостей.
В плане безопасности и междоменки важно помнить про CORS политика SOP запрещает запросы к другому домену поэтому сервер должен присылать правильные заголовки 
Access Control Allow Origin. Проблемы типа XSS внедрение скриптов лечатся санитизацией данных и политикой CSP а CSRF подделка запроса от имени юзера закрывается через анти CSRF токены и SameSite куки.
