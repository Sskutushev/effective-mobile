# Ответы на вопросы

### 1 Распространение событий
Это просто путь события по дереву дом у него есть маршрут туда и обратно. Фазы всего три.
Capturing это погружение когда событие идет сверху вниз от window до предка цели. Чтобы поймать тут используем флаг capture true.
Target это сама цель когда событие приземлилось на элемент по которому кликнули event.target.
Bubbling это всплытие самая рабочая тема когда событие поднимается обратно к window и на этой фазе обычно все и ловится.

На практике это нужно для делегирования событий когда вешаем один обработчик на родителя вместо ста на кнопки. Это экономит память и работает с динамическими элементами. Плюс понимание фаз помогает разруливать кейсы через stopPropagation и различать target кто кликнул и currentTarget где висит код.

### 2 Промисы и цикл событий
Промисы это объекты обещания асинхронного результата. Если он уже решился его нельзя отменить или передумать состояние фиксируется намертво. 
Три состояния это Pending ждем Fulfilled успех и Rejected ошибка. 
Обработка идет через then catch и finally но по факту сейчас все юзают async await это просто сахар над промисами чтобы код был плоским и чистым. 

Цикл событий это механизм который разруливает выполнение кода и позволяет js не блокировать поток на тяжелых задачах. Его главная работа постоянно чекать стек вызовов Call Stack и когда стек пустой он закидывает туда задачи из очередей. 
У промисов приоритет выше пока очередь микрозадач не пуста цикл событий не перейдет к обычным задачам типа таймеров или кликов.

### 3 ООП в JS
ООП это подход где представляем все в виде объектов которые взаимодействуют между собой. Помогает собирать код как конструктор из логических кирпичей.
Основные принципы это инкапсуляция когда прячем логику внутри наследование когда создаем новый класс на базе старого полиморфизм когда один метод работает по разному для разных типов и абстракция когда скрываем детали реализации оставляя только важное.

В js это реализовано специфично через прототипное наследование. Когда ищем свойство у объекта js сначала смотрит у него самого потом лезет в его прототип и так далее по цепочке. 
Синтаксис class это сахар над прототипами под капотом все равно остаются функции конструкторы и объекты. Классы делают код читабельнее а понимание принципов помогает строить нормальную масштабируемую архитектуру.

### 4 Обработка URL и рендеринг
Когда вводишь URL браузер превращает текст в пиксели за несколько этапов.
Сначала идет навигация и DNS поиск ip адреса домена потом ставится tcp соединение и tls рукопожатие для https. 
Дальше сервер отдает html и браузер запускает Critical Rendering Path строит DOM и CSSOM деревья объединяет их в Render Tree считает геометрию в Layout и рисует пиксели в Paint.

Чтобы все летало юзают протоколы HTTP/2 или HTTP/3 QUIC и CDN чтобы забирать статику с ближайшего сервера. Оптимизируют рендеринг через минификацию сжатие картинок и юзают Vite для кэширования зависимостей.
По безопасности CORS и политика SOP запрещает запросы к другому домену поэтому сервер должен слать правильные заголовки. XSS лечится санитизацией данных а CSRF закрывается через токены и samesite куки.
